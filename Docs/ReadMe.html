Introduction

PasHi is a simple command line program that highlights Pascal source code. It reads the original source code from standard input or the clipboard and writes the highlighted code to standard output or the clipboard. XHTML is used for the highlighted code and cascading style sheets are used to do the styling.

PasHi can either generate complete XHTML documents or just fragments of XHTML code containing the highlighted source. Complete XHTML documents contain an embedded CSS style sheet while code fragments require the user to provide the required CSS classes. This last feature makes it easy to embed code highlighted by PasHi into exiting web pages with highlight colours defined by the web page author.

A beta version of PasHiGUI, a GUI front end for PasHi, is included in the release.

Up until and including v1.0.0, PasHi was known as PasH.

Why another Pascal highlighter?

Simple, I wanted an easy to use tool to format Pascal code for the DelphiDabbler website. I wanted the following:

  1. Cascading style sheets.
  2. Valid XHTML.
  3. Full control over the appearance of the output code.
  4. Ability to highlight via the clipboard.
  5. The option to generate XHTML fragments for inserting into existing documents.

On checking the available free tools I've found they were either too complex or didn't provide one of the features I wanted. I'd already written highlighter code for my CodeSnip program, so I extracted that, tweaked it and wrapped it up in a command line program. Having made it I thought I'd release it in case anyone else finds it useful.

Check these two sites for examples of PasHi in use: Rudy's Delphi Corner and Hallvard's Blog. PasHi has also been used to highlight example source code on the DelphiDabbler Delphi Tips and Articles pages.

Installation

PasHi's simplicity means it that has no dependencies and creates no config files or registry entries. So, I haven't provided an installer, just a zip file containing a few files. They are:

  • PasHi.exe
    This is the program's executable file. Just copy it to a directory of your choice. Then open a command console and run PasHi. See Using PasHi below for details of PasHi's command line switches. You'll find PasHi more usable if you place it somewhere on your path or if you add PasHi's directory to the path.
  • PasHiGUI
    The GUI front end for PasHi. If you want to use this program just copy it to the same directory as PasHi.
  • License.txt
    Contains the program's license. Please read this and only use PasHi (and PasHiGUI if you agree with it. OK, given that you probably won't read I'll tell you now it's a pretty permissive open source license. You can use and distribute PasHi freely and get the source code and modify it. If you release your changes there are two conditions: (1) you must make the source available under the Mozilla Public License and (2) you must delete the program icons or use your own.
  • ReadMe.html
    This file. Nuff said.

To uninstall, just delete Pashi.exe and PasHiGUI.exe. Neither program makes any changes to your system.

Using PasHi

By default PashHi reads input from standard input and writes highlighted output to standard output. You can process a single file and write output to another file using shell redirection. For example, to highlight source.pas and output to source.html use: 

PasHi <source.pas >source.html

Note that if you do not redirect output the resulting HTML is displayed on the console. If you don't redirect input then you must enter the source code manually on the command line a line at a time and press Ctrl+Z and Return when you have finished to signify end of file.

PashHi's behaviour can be controlled using a parameter list. Options are included to change input and output locations and for controlling output format.

PasHi's usage is (all on one line): 

PasHi
   [<filenames> | -rc]
   [-o <filename> | -wc]
   [-frag]
   [-nobrand]
   [-style <filename>]
   [-linkcss <url>]
   [-hidecss]
   [-lang <language-code>]
   [-title <text>]
   [-e <encoding>]
   [-q]

or 

PashHi -h

The command line options are:

<filenames>
Specify one or more input (source) file names. Source code is read from each file name in turn and concatenated before being highlighted. Standard input is ignored. Specifying one file name has the same effect as redirecting standard input from that file. Do not provide file names in combination with the -rc switch: behaviour may be unpredictable.
-rc
Reads source code from the clipboard instead of standard input. Do not specify any file names on the command line when using this switch: behavour may be unpredictable.
-o <filename>
Write highlighted code to the file specified by the following file name parameter instead of to standard output. Using this switch has exactly the same effect as redirecting standard output to a file. Do not use this switch in combination with the -wc switch: behaviour may not be as expected.
-wc
Writes highlighted code to the clipboard instead of standard output. All output is stored as Unicode and complete XHTML documents have no charset information recorded in the file. Do not use this switch in combination with the -o switch: behaviour may not be as expected.
-frag
Outputs a XHTML code fragment containing just the highlighted code contained in a <pre> ... </pre> tag pair. This option is designed for pasting code fragment into an exiting XHTML document. This is a common requirement when adding highlighted code to a web page. The web page will need to provide suitable CSS code to style the fragment – see below for details of the required CSS classes. The -lang, -style, -linkcss, -hidecss and title switches are ignored when -frag is used.
-nobrand
Switch off the output of the comment included in code fragments that declares the the code was generated by PasHi. This switch works in combination with the -frag switch and is ignored if -frag is not also specified.
-lang <language-code>
Specifies language used in output document. A language code recognised by HTML must be provided in the parameter that immediately follows the switch. The code is not checked. If this switch is not supplied no language information is included in the output file. This switch should not be combined with the -frag switch.
-title <text>
Specifies title of output document. The required title must immediately follow the switch on the command line. Titles containing spaces must be enclosed in double quotes. This switch has no effect if the -frag switch is also used.
-style <filename>
Specifies the name of a file that contains a CSS style sheet that is to be embedded in the output document. The file name must be given in the parameter immediately following the switch. The CSS file must define the CSS styles described below. This switch is ignored if the -frag switch is also specified.
-linkcss <url>
Specifies the URL of an external CSS file linked to be the generated XHTML file. The URL parameter must immediately follow the switch. The URL can be relative or absolute. The linked CSS file must define the CSS styles described below. Since no CSS is embedded in the output file the -hidecss switch does nothing.
-hidecss
Complete XHTML documents output by PasHi may have the necessary CSS classes contained in a <style> section in the document header. The CSS is not protected by HTML comment tags (<!-- ... --> by default. This may cause problems for older browsers which may render the CSS as text. If you have this problem use this switch to enclose the CSS in HTML comments. The switch is ignored when the -frag or -linkcss switches are used.
-e <encoding>
Sets the character encoding used for output. The switch must be immediately followed by an encoding parameter. XHTML documents are marked with the chosen format and the resulting file may include byte order marks. This switch is ignored if -wc is specified, because data is always written to the clipboard in Unicode. Valid encoding parameters are:
  • utf-8 or utf8: encodes in Unicode using UTF-8 format with byte order mark.
  • utf-16, utf16 or unicode: encodes in Unicode using UTF-16, little endian format with byte order mark.
  • windows-1252, windows1252, latin-1 or latin1: encodes in Windows-1252 format.
  • iso-8859-1: encodes in ISO-8859-1 format.
-q
Specifies "quiet mode". PasHi does not write output to the console. Note that PasHi writes console output to standard error since standard output can be used to output the highlighted source code. This switch is ignored when the -h switch is used or if PasHi detects command line errors.
-h
Displays the help screen. The remainder of the command line is ignored.

If PasHi exits normally then 0 is returned to the OS as an exit code. In case of error the exit code is 1.

Perhaps the most common usage of PasHi is to read source code from the clipboard and replace it with a highlighted fragment. Here's what I do when pasting highlighted source into a web page:

  1. Select the unformatted source in an editor and copy it to the clipboard.
  2. Switch to a command window and do 
    PasHi -rc -wc -frag
    .
  3. Switch to the editor containing the web page and paste in the highlighted code.

CSS styles used by PasHi

The HTML produced by PasHi uses the following CSS classes:

.pas-source
Used to provide the name and size of font for the whole source code. You could also specify a background colour here. Alternatively use a containing <div> to supply the background.
.pas-comment
Style for comments.
.pas-kwd
Style for keywords and reserved words.
.pas-ident
Style for non-keyword identifiers.
.pas-sym
Style for symbols (e.g := and >).
.pas-str
Style for string literals.
.pas-num
Style for whole numbers (integers).
.pas-float
Style for floating point numbers.
.pas-hex
Style for hexadecimal numbers.
.pas-preproc
Style for preprocessor directives.
.pas-asm
Style for assembler.
.pas-space
Style for whitespace.
.pas-err
Style for any errors detected.

All classes excpet .pas-source just define text colour, style and weight.

Whenever you need to define your own CSS style sheets to use with files generated by PasHi you must provide definitions for all the above styles. You will need your own CSS if you:

  • Embed your own styles in the output XHTML document using the -style switch.
  • Link to an external style sheet from the output document using the -linkcss switch.
  • Include a code fragment generated by the -frag command in another HTML file.

To help with this, here are the default definitions for the classes that are used by PasHi: 

.pas-source {
    font-family: "Courier New", monospace;
    font-size: 9pt;
}
.pas-comment {
    font-style: italic;
    font-weight: normal;
    text-decoration: none;
    color: #000080;
}
.pas-kwd {
    font-style: normal;
    font-weight: bold;
    text-decoration: none;
}
.pas-str {
    font-style: normal;
    font-weight: normal;
    text-decoration: none;
    color: #800080;
}
.pas-num, .pas-float, .pas-hex {
    font-style: normal;
    font-weight: normal;
    text-decoration: none;
    color: #800000;
}
.pas-preproc {
    font-style: normal;
    font-weight: normal;
    text-decoration: none;
    color: #008000;
}
.pas-asm {
    font-style: italic;
    font-weight: normal;
    text-decoration: none;
}
.pas-err {
    font-style: normal;
    font-weight: normal;
    text-decoration: none;
    color: #FF0000;
}
.pas-sym, .pas-space, .pas-ident {
}

Change the styles to suit your needs.

Using PasHiGUI

PasHiGUI is very simple and pretty straightforward. A brief explanation of the commands and main display should suffice.

Commands

File | Open
Opens a source file for highlighting.
File | Save
Saves highlighted code as XTHML to a file. The file can be saved with either an html or txt extension.
File | Exit
Quits the program.
Edit | Copy
Copies highlighted code to the clipboard as XHTML in text format.
Edit | Paste
Attempts to highlight any text on the clipboard: it assumes it is source code.
Options | Generate Fragment
When this option is set an HTML fragment is generated instead of a complete XHTML document. For more information see details of PasHi's -frag switch.
Options | Show Hints
Determines whether help hints are displayed in the status bar as the mouse moves over menus and other parts of the interface. Turn this on if you want to be reminded of a command etc.
Help | About
Displays the About Box. Useful to find the version number of the program.

Main Display

There are three display tabs each of which render the generated XHTML in a different way:

Display View
Renders the XHTML as it will appear in a browser. If the generated code is a complete document it is display as-is. If it is a fragment the code is displayed in a yellow box.
HTML View
Displays the generated XHTML code.
Source Code View
Displays the original source code in plain text.

Errors

If you see an error telling you that there is an error executing PasHi because "the system cannot find the file specified" this means that PasHiGUI can't find PasHi. Make sure that PasHiGUI is placed in the same directory as PasHi.

You may see an error if PasHi is the wrong version. Make sure PasHi's file version is at least v1.0.1 and the product version is at least v1.1.0. If you installed both programs from the same zip file you won't have this problem.
